构建异步 action 对象

  • 普通 action 为 普通的 js 对象， 而异步 action 对象为一个函数:

    总体来 异步 action 对象本质就是在发送 axios 请求中穿插发送不同 type 的 action

    ​

如何处理 action 对象？

  • 我们利用中间件 redux-thunk ，中间件位于 action 与reducer 之间。
  • redux-thunk 用来筛选 action 类型。 如果为 js对象，直接放行，如果是函数，则进行处理再放行

使用 redux-thunk

在to8to项目中已经设置好中间件的使用，请不要修改文件

reducer 处理

利用 status 获取请求的状态， 其他就正常逻辑

请求成功的话在 store 里就可以有数据了。

在to8to项目中使用

使用 axiosGetAction, axiosPostAction, axiosStatus, axiosActionTypes 即可

axiosGetAction 为 axios get 方法

axiosPostAction 为 axios post 方法

axiosStatus 为 axios 的请求的三种状态 loading, success, failure

axiosActionTypes 为 axios 请求的共用 action
