《其他》Less，Sass，Stylus之间的差异

2018-01-21

CSS轮子争夺战—LESS SASS STYLUS

​ 对于前端工程师来说，一个好的css预处理器可以帮助我们更好的提高工作效率。所以说选择一个适合的css预处理对于我们来说也是一件值得考虑的事情。

​ 本文主要对目前流行的三大css处理器：Less、Sass、Stylus进行一些方面的对比，从而在其中选择一款适合自己的css预处理器

1.背景介绍

Sass：

​ Sass诞生于2007年，是最早最成熟的一款CSS预处理器语言，它可以适用变量、常量、嵌套、混入、函数等功能。

​ 目前Sass有两套语法规则：

​ 1：利用缩进作为分隔符来区分代码块

​ 2：采用 ({}) 作为分隔符

Less：

​ Less产生于2009年的一个开源项目，受Sass影响比较大，采用css语法，能够很容易的让大部分开发者和设计师上手、Less提供了很多方式能平滑的将写好的代码转化成标准的css代码，在很多流行的框架和工具中已经能经常看到LESS的身影了（例如Twitter的Bootstrap框架就使用了LESS）

Stylus：

​ Stylus被称为是一种革命性的新语言，提供一个高效、动态、和使用表达方式来生成CSS，以供浏览器使用。Stylus同时支持缩进和CSS常规样式书写规则。
