let 相关

let为块级作用域，声明时就绑定在了所属的块级内，在循环体中则作用域在循环体内。

  • 不存在变量提升

    在使用 var声明变量时，在声明前使用变量，该变量的值为 undefined

    要想改变这种规则，可以使用严格模式

    let则改变了这个规律，变量在没定义前使用，会直接报错

  • 暂时性死区

    暂时性死区规定了在let声明就已经绑定了一个局部变量。let遵守着没声明前使用，则会报错。

    在以上例子里虽然已经存在了全局变量tmp，但是在if判断里let声明了同名变量，所以if会先绑定let声明的变量。但是在let声明前使用了该变量，所以系统会报错

    • 因为暂时性死区的原因 typeof 返回的类型信息也不一定准确

    隐蔽的暂时性死区

    例子因为函数bar的参数x默认了为y，但此时y还没有被声明，所以会报错

    和var不同的报错

    因为关键字var在声明变量前系统内部会实现var x = undefind，所以var x = x会不报错。但是在let这里便不通过了。

    • 暂时性死区总结：凡是在let声明前使用了let变量的，就会报错

  • 不允许重复声明

    • 在同一作用域内不许重复声明同一个变量
    • 在函数体内部同样不许重复声明同一变量

    块级作用域

在es6之前，js只有全局作用域和函数作用域。这样很容易发生变量污染和泄漏。其实早在N年前，js就退出了块级作用域，但是因为改革对当时的大厂冲击比较大，就一直搁浅到现在。终于在es6中采用了块级作用域。

  • ES6允许会级作用域的任意嵌套

    内部可以读取外部声明的变量，外部不可读取内部

  • 内层作用域可以定义外层作用域的同名变量

  • 块级作用域的出现，实际上使得获得广泛应用的立即执行函数表达式（IIFE）不再必要了。

    • IIFE 函数自调用，为了防止变量污染，类似闭包

  • 块级作用域与函数声明

    在 es5 中，规定函数只能在顶层作用域和函数作用域之中声明，不能在块级作用域声明。

    在 es6 中，明确规定允许在块级作用域中声明函数。

    es6 规定，块级作用域之中，函数声明语句的行为类似let ,在块级作用域之外不可引用。

const 相关

  • 基本用法

    const声明一个只读的常量。声明之后，常量的值就无法改变。

    只声明不赋值，会报错

  • 作用域

    const 与 let 作用域相同，都是块级作用域。只在声明所在的块级作用域内有效。

    const 也会存在暂时性死区

  • 本质

    本质上const 保存的不是变量的值，而是变量指向的那个内存地址。

    但是如果 const 指向的内存地址保存的是一个对象的话，const 只能保证指向内存地址不变，但是对象的内部结构就控制不了了。

    如果要解决这个问题可以适用如下方法：

    • Object.freeze 对象冻结

顶层对象与全局变量

在es6之前，声明的全局变量都与顶层对象相挂钩。

在浏览器环境中，顶层对象为window

js的这个设计模式被认为是JavaScript语言最大败笔之一。

但是在es6中，把全局变量与顶层对象之间挂钩取消了。

这样 let 声明的变量与顶层对象 window 并没有直接联系。
